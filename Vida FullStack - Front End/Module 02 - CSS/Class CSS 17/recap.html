1. Make a visually pleasing page;
2. Stick to a consistant theme accross the whole page;
3. Provide minimal usability to the page;
4. Use a little bit of knowledge of each of the 17 classes to make the page;
5. Use external resources to apply extra stylings to the page;
6. Don't overdo and leave the code as clean as possible;

State of the Realm (Overview of the Module)

Starting this review/overview of the module, it has to be said that it is a long module, spanning hours of content to work with. Most of the classes are straightforward and simple, though I take my time as I want to properly document everything on my personal Notion Page. I took around 25 days to complete the class, with the entire annotation system and some delays.

Reviewing a bit class by class:
> Class 01 served as an introduction to CSS, bringing the basics of the style tag.
> Class 02 was about the differences of External CSS, Internal CSS and Inline CSS. In this code, I tried to keep most of the CSS on an external style.css file, but to use a bit of the knowledge, I did an inline application.
> Class 03 explained the selectors, being them the tags, classes or IDs.
> Class 04 was about specificity, albeit a bit abstract right now, it seemed simple to understand. It's somewhat strange to think about it. I believe it is very present in this code, but I still need to work with more complex situations to properly get a grasp on it.
> Class 05 was about Inheritance, much more straightforward concept and widely used here.
> Class 06. Colours. Not much to say here. Albeit a very simple concept, there are many different ways to use it. In the class, we also took a look at some online colour palette websites.
> Class 07 went over different measuring units to use on styling. I think it is a very straight forward lesson and pretty simple to understand. Using it in this practice has been a bit sloppy, but still manageable.
> Class 08 was super simple, about margins, paddings and borders in general. Very important concepts.
> Class 09 was simple yet complicated to apply. It went over display visibility, but trying to use it in this webpage proved to be quite complicated and unstable. I need to train much more on this.
> Class 10 on positioning was simple, but using it together with Display Visibility in this page did quite a number on me.
> Class 11 about border radius was as specific as you can be. It was fun, and I can see much of the use here. Also in this class as a little extra, we played a bit with the :hover selector property, very interesting.
> Class 12 on backgrounds, simple, but efficient. Also I had some problems doing the background here, but at the end of the day, they were more related to the image I chose to add here than the proper application. I learned a lot of strategies for background doing this recap.
> Class 13 added gradients to the background, and allowed me to add a bit more flair to the pages. I used a lot of it here.
> Class 14. Fonts. I love fonts. I loved learning to add fonts to the page. The only thing we didn't go over was how to add completely custom fonts that are not on Google Fonts, but I'm sure I'll go over it someday.
> Class 15 was about the applications and transformations of Text. Simple and powerful. I used a lot of it, specially on the links.
> Class 16, nearing the end, was over shadows. Although very cool, I think it's something that needs to be used with caution, to not be overdone.
> Class 17 was the final class, about making a responsive design for different screen sizes. Very important nowadays. I'm adding this last on the CSS of this page, I don't know if this is the proper protocol or if I should've been doing it as I went. It didn't really work out in this page, but it is something that I'll keep my mind to the next time.
> Class 18 is this recap finally. Not an official class, but something I like to make to test what I learned. I did it with the HTML module and I'll keep doing it when it's relevant.
